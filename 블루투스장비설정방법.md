# 실제 블루투스 하드웨어 연동을 위한 인터페이스 가이드

현재 `FaceHitmap` 웹 어플리케이션은 **Web Bluetooth API**를 기반으로 하드웨어 수분 측정 장치와 연결하도록 설계되어 있습니다. 
아두이노(Arduino)나 펌웨어 등 실제 센서 측정 장비와 본 웹 시스템이 실시간으로 데이터를 통신하기 위해서는, 아래 세 가지 인터페이스 규약을 사전에 맞춰야 합니다.

---

## 1. BLE 서비스 UUID (Service UUID) 설정

스마트폰이나 PC 브라우저가 측정기를 찾을 때 사용하는 "고유 식별 번호"입니다.

* 하드웨어 장치의 BLE 서비스 UUID를 프론트엔드 코드에 등록해주어야 브라우저가 장치를 발견하고 페어링할 수 있습니다.
* 해당 UUID는 `src/hooks/useBluetoothDevice.ts`의 `navigator.bluetooth.requestDevice` 함수 내 `optionalServices` 배열에 추가됩니다.
* **장비팀(하드웨어 개발자)에게 전달받아야 할 정보:** "수분 데이터 발신에 사용되는 GATT Service UUID" (예: `12345678-1234-5678-1234-56789abcdef0` 또는 표준 `0x181A`)

## 2. 데이터를 읽기 위한 Characteristic UUID 파악

수분 측정기와 성공적으로 연결된 후 실제 센서 측정값을 실시간으로 수신(Subscribe/Notify)하기 위해서는 특정 채널(Characteristic)을 구독해야 합니다.

* 하드웨어 장치에서 측정값을 발행해주는 Characteristic UUID를 알아야 합니다.
* **장비팀(하드웨어 개발자)에게 전달받아야 할 정보:** "센서 데이터를 구독(Notify/Indicate)하기 위해 어느 Characteristic UUID를 연결해야 하는가?"

## 3. 데이터 송수신 포맷 (Payload Format) 일치

사용자가 장비로 측정을 완료했을 때 웹 브라우저로 전송되는 데이터 배열(바이트 Payload)을 자바스크립트로 어떻게 파싱할지에 대한 규약입니다.

* 전송되는 **바이트 배열(DataView)**의 순서 및 의미를 알고 있어야 합니다.

### 권장하는 데이터 페이로드(Payload) 예제:
* **Byte 0 (1바이트):** 피부 측정 부위 ID
  * 0: 이마 (Forehead)
  * 1: 왼쪽 볼 (Left Cheek)
  * 2: 오른쪽 볼 (Right Cheek)
  * 3: 턱 (Chin)
  * 4: 코 (Nose)
  * 5: T존 (T-Zone)
* **Byte 1 (1바이트):** 수분량 수치 (0 ~ 100 퍼센트)

### 실제 코드 적용 예시:
이 프로토콜이 합의되면 프론트엔드의 데이터 수신 콜백 리스너를 아래와 같이 간단하게 구현할 수 있습니다.

```typescript
// src/hooks/useBluetoothDevice.ts 내부 구현 예시

const handleCharacteristicValueChanged = (event: any) => {
    // 기기에서 전송받은 2바이트 바이너리 데이터 읽기
    const dataView = event.target.value; 
    
    const regionId = dataView.getUint8(0);
    const sensorValue = dataView.getUint8(1); 
    
    // 부위 ID 사전 정의
    const regionMap: Record<number, string> = {
        0: 'forehead',
        1: 'left_cheek',
        2: 'right_cheek',
        3: 'chin',
        4: 'nose',
        5: 't_zone'
    };
    
    const regionName = regionMap[regionId];

    if (regionName && sensorValue >= 0 && sensorValue <= 100) {
        // 리액트 컴포넌트 상태 업데이트 -> UI에 실시간 수분량 반영됨
        setMeasurementData({ region: regionName, value: sensorValue });
    }
};
```
