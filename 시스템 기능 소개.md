# 글로벌 타겟 3단계 피부 분석 및 맞춤형 성분 매칭 파이프라인 계획서

본 문서는 인종, 연령, 기후 등 후천적 환경 요인(Exposome)과 기기 및 설문 데이터를 접목하여 글로벌 스텐다드(바우만 피부 분류 등)에 부합하는 체계적인 피부 분석 및 화장품 매칭 기능을 구현하기 위한 계획서입니다.

## User Review Required
* **하드웨어 센서 프로토콜 스펙**: 기존 블루투스 모듈에서 하드웨어 연동 시, 각 측정치(정전용량, 임피던스 등)를 스케일링하여 0~100 사이의 % 수치로 가공해서 올려주는 역할을 앱에서 할지/하드웨어에서 할지 정리가 필요할 수 있습니다. (현재는 `[0~100]` 값으로 들어온다고 가정)
* **설문 시스템 항목**: 바우만 피부 유형 설문(BSTI)을 차용한 글로벌 자가 진단 설문의 문항(문항 갯수 등)을 어느 정도로 간소화할 건지 기획 확정 필요

## Proposed Changes

---

### Phase 1. 피부 데이터 수집 모듈 (Multi-Modal Data Collector)

#### 1-1. 팝업(Modal) 기반 프로필 및 설문 수집 UI (`Dashboard.tsx`, `SurveyModal.tsx`)
- 카메라나 하드웨어 연동 전에 팝업(Modal)을 띄워 사용자 프로필(나이, 인종, 거주 기후)을 필수 입력받도록 구현합니다.
- 기기 측정이 불가능한 사용자를 위한 글로벌 자가 진단 설문 인터페이스(바우만 피부 유형 설문 벤치마킹 요소: 건조감, 홍반, 색소침착 수준 자가평가 문항)를 팝업 내 스텝(Step)으로 추가합니다.

#### 1-2. 다양한 측정 방식의 인터페이스 추상화 (`useBluetoothDevice.ts` / `useSkinAnalysis.ts`)
- 기존 `useBluetoothDevice`에 더해 (1) **하드웨어 센서 연동 (기존+추가 확장)**, (2) **소프트웨어/AI 기반 웹캠 분석 (사진 기반 모공/색소침착 판단)**, (3) **설문 데이터 기반 분석**을 모두 `hydrationData` 형태의 중간 인터페이스로 매핑해주는 파이프라인을 구축합니다.

---

### Phase 2. 글로벌 타겟 피부 유형 분류 모듈 (`recommendations.ts` / 분류 엔진 심화)

1차원적 수치 비교를 벗어나 **선천적 요인 + 후천적 환경 요인**을 결합한 2-Stage 분류 로직을 구현합니다.

#### 2-1. 선천적 유분 중심의 1차 분류 로직
- T존과 U존의 센서값(유분 및 수분) 또는 모공 카메라 분석 결과를 결합하여 선천적 피지 분비 능력을 판별합니다.
- 결과 분류: **Alipic(피지가 적은 건성)**, **Normal(중성)**, **Oily(지성)**

#### 2-2. 후천적 상태 및 환경 요인(Exposome) 반영한 2차 분류 로직
- 나이(각질층 수분 함유량 감소), 인종(백인vs동양인 노화/색소 양상 차이), 기후 환경(습도/자외선) 정보를 곱하여 보정값(Weight)을 산출합니다.
- 결과 분류: 표피 건조감 유무, 민감도(홍반 발생 쉬움), 색소침착 위험군 유무 판별.

---

### Phase 3. 베이스 + 활성 성분 결합 화장품 매칭 파이프라인

분류된 피부 유형(1차+2차)을 토대로 화장품 제형 및 핵심 기능성 성분을 도출합니다.

#### 3-1. 제형(Base) 선택 및 활성 성분(Active) 조합 엔진 (`ReportView.tsx` & `recommendations.ts`)
- **베이스 파운데이션 추천**: 1차 분류 결과에 따라 제공
  - Alipic -> 크림(Cream), 밤(Balm) 등 리치한 제형 추천
  - Normal -> 로션(Lotion), 마일드 크림 추천
  - Oily -> 젤(Gel), 세럼(Serum) 등 가벼운 제형 추천
- **활성 성분 추천**: 2차 분류(후천적 요인)에 따라 타겟티드 성분 추가
  - 예: 민감도 높음 -> 병풀, 알란토인
  - 예: 색소침착 높음 + 백인 타겟 혹은 기후 -> 비타민 C 유도체, 나이아신아마이드
  - 예: 노화(40대 이상 건성) -> 레티놀, 펩타이드 (단, 민감도가 높으면 대체 성분 바쿠치올 등 매칭)

#### 3-2. Gemini AI 고도화 (`GeminiService.ts`)
- 프롬프트 엔진 모델을 **Gemini 최신 버전(예: `gemini-1.5-pro` 또는 사용 가능 최신 모델)** 로 업그레이드합니다.
- "측정 데이터, 1/2차 분류된 타입, 베이스 제형 및 추천 성분"의 모든 컨텍스트를 던져주고, 환자에게 처방전을 쓰듯 전문가 포맷으로 자연어 분석 결과를 내도록 프롬프트를 치밀하게 구조화합니다.

## Verification Plan
1. `Dashboard` 접속 시 모달 팝업이 떠서 나이, 인종, 지역, 자가 진단 설문(선택)을 받는지 확인합니다.
2. 기기 데이터로 시뮬레이션을 실행하여 T존, U존 별도 처리 로직 기반으로 측정 완료합니다.
3. `ReportView`에서 단순 수부지/지성이 아닌 (1차 Base Type) + (2차 상태) 결합 문자열(예: Oily-Sensitive)이 노출되는지 확인합니다.
4. 화장품 추천 파트에 제형(텍스처) 정보와 목적별 활성 성분(Active) 리스트가 분리되어 깔끔하게 제공되는지 확인합니다.
5. 최신 Gemini 모델이 풍부한 전문 처방(처방전 톤앤매너 등)을 생성해 주는지 검증합니다.

---

### Phase 4. 사용자 프로필 데이터 영구 저장 및 수정 기능 구현 (추가 요구사항)

사용자가 매번 나이/인종/설문을 새로 입력하는 번거로움을 줄이기 위해 **브라우저 로컬 스토리지(`localStorage`)**를 활용하여 프로필 데이터를 보관합니다.

#### 4-1. 데이터 저장 및 초기 로드 로직 (`Dashboard.tsx`)
- 애플리케이션 진입 시 `localStorage.getItem('faceHitmapUserProfile')`를 확인합니다.
- 저장된 값이 없다면 기존처럼 `showSurvey`(설문 모달)를 즉시 활성화합니다.
- 저장된 값이 존재한다면, JSON 파싱 후 상태(`userProfile`)에 즉시 반영하고 모달을 띄우지 않은 채 메인 대시보드로 진입합니다.

#### 4-2. 사용자 정보 변경(수정) 버튼 UI 추가 (`Dashboard.tsx`)
- 대시보드 우측 상단이나 좌측 사용자 정보 영역 쪽에 **[사용자 프로필 수정]** 버튼을 새롭게 추가합니다.
- 해당 버튼 클릭 시 `showSurvey` 상태를 다시 `true`로 만들어 설문 모달(`SurveyModal.tsx`)을 재호출합니다.
- 이때, 모달 화면에 기존 폼 입력값(나이 50대, 건조감 8 등)이 원활히 복원(Prefill)되어 있어야 사용자가 일부 값만 변경하기 수월하므로, `SurveyModal`에 `initialData` 프롭(Prop) 전달 구조를 보강합니다.

#### 4-3. 데이터 갱신 및 유지보수
- 모달 내 **[진단 시작]** 또는 **[저장]** 버튼을 눌러 결과가 `onComplete`로 넘어오면, 최신값을 다시 `localStorage.setItem`으로 덮어씁니다.
