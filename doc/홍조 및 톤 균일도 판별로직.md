# 홍조 및 톤 균일도 판별로직

## 개요

피부 분석 시스템에서 **홍조(Redness)**와 **톤 균일도(Evenness)**를 판별하는 알고리즘의 설계 문서입니다.
CIELAB 색공간의 `a*` 채널(붉은기)과 `L*` 채널(밝기)을 기반으로 부위별 피부 상태를 분석합니다.

---

## 1. 색공간 변환

### RGB → CIELAB 변환 파이프라인

```
RGB → (감마 보정) → Linear RGB → XYZ (D65) → CIELAB
```

- **L***: 밝기 (0~100)
- **a***: 붉은기/초록기 축 (-128 ~ +127), 양수일수록 붉음
- **b***: 노란기/파란기 축 (-128 ~ +127)

---

## 2. 홍조(Redness) 판별

### 2.1 절대 홍조 점수

CIELAB `a*` 값에서 **동양인 피부 기준선(baseline)**을 차감한 후 정규화합니다.

```
기준선(baselineA) = 15  (동양인 평균 a* 값)
adjustedA = max(0, a* - baselineA)
redness = clamp(0, 100, (adjustedA / 20) * 100)
```

| a* 값 | 의미 | redness 점수 |
|-------|------|-------------|
| 10 | 정상 피부 (창백) | 0% |
| 15 | 동양인 평균 | 0% |
| 20 | 약간 붉음 | 25% |
| 25 | 눈에 띄게 붉음 | 50% |
| 35 | 심한 홍조 | 100% |

> **왜 기준선이 필요한가?**
> 동양인 피부의 a* 값은 정상 상태에서도 10~18 범위입니다.
> 기준선 없이 `(a*/30)*100`을 사용하면 정상 피부도 33~60%로 계산되어 거짓 양성(false positive)이 발생합니다.

### 2.2 상대 편차 점수 (부위 간 비교)

얼굴 전체 평균 `a*` 대비 해당 부위의 편차를 계산합니다.

```
avgA = 모든 부위의 a* 평균
aDev = max(0, 해당 부위 a* - avgA)
relativeScore = clamp(0, 100, (aDev / 8) * 100)
```

- 분모 `8`은 피부과학적으로 의미 있는 편차(ΔE ≈ 5~10)를 반영합니다.
- 이 점수가 높으면 **특정 부위만 유독 붉다**는 의미입니다.

### 2.3 상태 판정

절대 홍조와 상대 편차를 **복합 조건**으로 판정합니다.

```
Warning (경고): redness > 50 AND relativeScore > 50
Caution (주의): redness > 25 OR relativeScore > 60
Normal  (정상): 그 외
```

> **핵심 설계 결정**: Warning은 `AND` 조건입니다.
> 절대 홍조만 높거나 상대 편차만 높은 경우는 Caution까지만 판정합니다.
> 이로써 코, 볼 등 자연적으로 혈관이 밀집한 부위의 거짓 경고를 방지합니다.

---

## 3. 톤 균일도(Evenness) 판별

### 3.1 계산 방식

CIELAB `L*` 값의 **표준편차(σ)**를 기반으로 균일도를 산출합니다.

```
avgL = 모든 부위의 L* 평균
varianceL = Σ(Li - avgL)² / N
stdDevL = √varianceL
evenness = clamp(0, 100, 100 - (stdDevL * 5))
```

| stdDevL | 균일도 | 의미 |
|---------|--------|------|
| 0 | 100% | 완벽하게 균일 |
| 5 | 75% | 자연스러운 편차 |
| 10 | 50% | 약간 불균일 |
| 20+ | 0% | 심하게 불균일 |

### 3.2 해석

- **80% 이상**: 균일한 피부톤
- **60~80%**: 자연스러운 범위
- **60% 미만**: 색소 침착이나 잡티에 의한 불균일 가능성

---

## 4. 샘플링 방식

각 부위(이마, 좌볼, 우볼, 턱, 코, T존)에서 **10×10 픽셀 영역**의 평균 RGB를 추출합니다.

```
sampleSize = 10
영역: (landmark_x - 5, landmark_y - 5) ~ (landmark_x + 5, landmark_y + 5)
→ 총 100 픽셀의 RGB 평균을 CIELAB으로 변환
```
