# 피부 분석 데이터 수집 항목 고도화 제안

현재 `FaceHitmap` 애플리케이션은 센서 측정값(수분, 유분)과 사용자의 기본 인구통계학적 정보(성별, 나이, 인종, 기후, 주관적 피부 상태)를 수집하여 추천 알고리즘과 AI(Gemini)에 활용하고 있습니다. 

향후 AI 추론의 정확도를 높이고 개인화된 양질의 인사이트를 제공하기 위해 데이터베이스(`measurements` 컬렉션)에 추가로 수집할 의미 있는 항목들을 제안합니다.

## 1. 피부에 직접적인 영향을 미치는 생활 습관 (Lifestyle)
환경적 요인 못지않게 생활 습관은 피부 상태의 핵심 변수입니다.
*   **수분 섭취량 (Water Intake):** 하루 평균 수분 섭취량 (예: 1L 미만, 1~2L, 2L 이상)
    *   *이유:* 실제 센서가 측정한 수분량(`hydrationData.moisture`)과 체내 수분 섭취량 간의 상관관계 추론
*   **수면 시간 (Sleep Duration):** 일평균 수면 시간 (예: 5시간 미만, 5-7시간, 7시간 이상)
    *   *이유:* 피부 재생 및 피지 분비율(`hydrationData.sebum`)의 원인 분석에 결정적 요인
*   **스트레스 지수 (Stress Level):** 주관적 스트레스 수준 (1~5 척도)
    *   *이유:* 성인 여드름 및 민감성 피부 변동성 추적

## 2. 현재 스킨케어 루틴 및 사용 제품 (Current Skincare)
AI가 더 정교한 '대체/추가 제품'을 추천하기 위해 현재 상태를 알아야 합니다.
*   **사용 중인 주요 화장품 종류 (Product Types):** 선크림 단독, 토너+로션, 앰플/세럼 추가 등
*   **활성 성분 경험 유무 (Active Ingredients Usage):** 레티놀(비타민 A), 비타민 C, AHA/BHA 등 고효능 성분 사용 여부
    *   *이유:* AI 추천 루틴 생성 시, 기존에 적응된 성분인지 파악하여 자극적인 추천 방지 (특히 `sensitivity` 수치가 높을 경우)

## 3. 피부 고민의 구체화 (Advanced Concerns)
현재는 건조함/민감도/색소침착만 1~10 척도로 받지만, 해결하고 싶은 목표를 명확히 해야 합니다.
*   **가장 해결하고 싶은 주된 고민 (Primary Target):** 
    *   선택지: 탄력 저하/주름, 트러블/여드름, 홍조, 모공, 칙칙한 톤 등
    *   *이유:* 결과 리포트에서 `t.report.aiRecommendation` 내용을 생성할 때 최우선 초점을 맞추기 위함

## 4. 메타데이터 (Contextual Metadata) - 자동 수집 고도화
사용자가 직접 입력하지 않아도 앱이 수집할 수 있는 상황 단서입니다.
*   **측정 시간대 분류 (Time of Day):** 아침 기상 직후, 세안 직후, 오후 스킨케어 전 등
    *   *이유:* 피부 수분/유분은 하루 중 언제 측정했느냐에 따라 크게 달라집니다. (예: `routineTab`과 연계)
*   **기상 데이터 (Weather API 연동시):** 측정 당일의 실제 외부 온습도 및 UV 지수
    *   *이유:* `climate` 설문 항목은 일반적인 거주 기후일 뿐, 측정 당일의 계절적 변화를 담지 못하므로 외부 날씨 API 연동 시 강력한 데이터가 됩니다.

### 도입 우선순위 및 UI/UX 고려사항

설문(`SurveyModal.tsx`)이 너무 길어지면 사용자가 피로감을 느껴 **측정 자체를 포기(Drop-off)**할 수 있습니다. 

**추천 전략:**
1.  **필수 항목 (Phase 1):** 현재 모달에 *'가장 큰 피부 고민 1가지(Primary Target)'* 객관식만 추가.
2.  **선택 항목 (Phase 2):** 결과 리포트를 본 직후, 더 정확한 다음 스킨케어를 위해 *'생활 습관 및 사용 중인 성분'*을 입력하는 추가 프로필 탭 제공.
