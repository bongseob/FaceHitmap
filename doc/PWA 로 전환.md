# Face Hitmap PWA 전환 가이드

본 문서는 기존 Next.js 웹 애플리케이션을 모바일 웹앱(PWA)으로 전환하기 위해 적용된 기술적 변경 사항과 배포 지침을 정리한 문서입니다.

## 1. 개요
PWA(Progressive Web App)는 웹 기술을 사용하여 모바일 앱과 유사한 사용자 경험을 제공하는 기술입니다. 별도의 스토어 설치 없이 홈 화면 추가를 통해 독립적인 앱처럼 작동합니다.

## 2. 주요 적용 사항

### 2.1 기술적 기반 (Infrastructure)
- **라이브러리**: `@ducanh2912/next-pwa`를 도입하여 Next.js와 PWA 기능을 통합했습니다.
- **설정 (`next.config.ts`)**: 
  - `withPWA` 플러그인을 적용하여 빌드 시 서비스 워커(`sw.js`)가 자동 생성되도록 구성했습니다.
  - 개발 모드(`development`)에서는 PWA 기능을 비활성화하여 디버깅 편의성을 높였습니다.
  - Next.js 16의 Turbopack과의 호환성 에러를 해결하기 위해 `turbopack: {}` 설정을 추가했습니다.

### 2.2 앱 정체성 (App Identity)
- **Manifest (`public/manifest.json`)**: 앱 이름, 테마 색상, 실행 모드(`standalone`), 아이콘 경로 등을 정의했습니다.
- **Iconography**:
  - `icon-192x192.png`: 안드로이드 및 일반 설치용.
  - `icon-512x512.png`: 고해상도 디스플레이 및 스플래시 화면용.
  - `apple-touch-icon.png`: Safari 홈 화면 추가용 아이콘.
- **메타 태그 (`src/pages/_document.tsx`)**: iOS 및 안드로이드 브라우저가 앱으로 인식할 수 있도록 `apple-mobile-web-app-capable` 등 필수 태그를 삽입했습니다.

### 2.3 UI/UX 최적화 (Mobile First)
- **반응형 레이아웃**: 대시보드, 리포트, 설문 화면의 간격과 크기를 모바일 세로 화면에 맞춰 재조정했습니다.
- **터치 인터랙션**: 버튼 클릭 시 `scale-95` 효과를 부여하여 물리적인 피드백을 강화했습니다.
- **이미지 미러링**: 셀카 촬영 시에는 거울 모드(좌우 반전)를 유지하고, 갤러리 사진 업로드 시에는 원본 그대로 표시되도록 로직을 구분했습니다.

## 3. 배포 시 주의 사항 (필수)

### 3.1 HTTPS 보안 연결
- PWA는 보안상의 이유로 **반드시 HTTPS** 환경에서만 설치 버튼이 활성화됩니다.
- 서버 배포 후 SSL 인증서(Let's Encrypt 등) 적용이 필수적입니다.

### 3.2 서비스 워커 업데이트
- 코드가 수정되어 배포될 때마다 브라우저는 서비스 워커를 통해 새 버전을 감지합니다. 
- 때때로 강제 새로고침(Ctrl+F5)이 필요할 수 있습니다.

## 4. 사용자 가이드 (안내용)
- **Android**: 사이트 접속 시 하단에 나타나는 "홈 화면에 추가" 배너를 클릭합니다.
- **iOS (Safari)**: 하단 중앙 '공유' 아이콘 클릭 -> '홈 화면에 추가' 선택.
