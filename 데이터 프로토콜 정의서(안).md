# 블루투스 수분 측정기 데이터 프로토콜 정의서 (초안)

**문서 상태:** 초안 (Draft)
**목적:** 하드웨어 장비(피부 수분 측정 센서)와 스마트폰/PC 웹 애플리케이션 간의 Bluetooth Low Energy(BLE) 통신 규격 정의
**버전:** v0.1

---

## 1. 통신 기본 규격 (Bluetooth LE)

| 항목 | 설정값 (안) | 비고 |
| :--- | :--- | :--- |
| **Service UUID** | `0000181A-0000-1000-8000-00805f9b34fb` | 환경 센싱 서비스(Environmental Sensing) 표준 UUID 사용 권장, 또는 커스텀 UUID 사용 가능 |
| **Characteristic UUID** | `00002A6E-0000-1000-8000-00805f9b34fb` | 온도/습도 특성 표준 또는 커스텀 UUID. 반드시 `Notify` 혹은 `Indicate` 권한을 활성화해야 함 |
| **통신 방식** | `Notify` (Server -> Client) | 기기에서 측정 완료 시 웹 프론트엔드로 즉시 데이터 푸시 |
| **Connection Interval** | 30ms ~ 50ms | |

## 2. 데이터 페이로드(Payload) 구조 설계

웹 프론트엔드로 전송되는 데이터는 실시간성과 파싱의 단순함을 위해 최소한의 바이트 배열(Byte Array)로 구성합니다.

### 2.1 패킷 구조 (총 5~6 Byte 권장)

기기에서 측정을 성공적으로 마치고 버튼을 뗄 때, 아래의 배열을 Characteristic에 Write(Notify)합니다.

| Byte Index | 명칭 | 타입 | 값의 범위 | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| `[0]` | **Header** (옵션)| `Uint8` | `0xAA` | (선택적) 패킷의 시작 |
| `[1]` | **Region ID** | `Uint8` | `0x00` ~ `0x05` | 측정 대상 피부 부위 (순차적 측정 시 무시 가능) |
| `[2]` | **Moisture (수분)** | `Uint8` | `0x00` ~ `0x64` | 측정된 수분 퍼센티지 (0 ~ 100%) |
| `[3]` | **Sebum (유분)** | `Uint8` | `0x00` ~ `0x64` | 측정된 유분 퍼센티지 (0 ~ 100%) |
| `[4]` | **Status** | `Uint8` | `0x00` ~ `0x01` | 정상(0x00), 에러(0x01) |
| `[5]` | **Checksum** (옵션)| `Uint8` | - | (선택적) 무결성 검증용 |

### 2.2 피부 부위 식별자 (Region ID) 매핑표

기기에서 사용자가 어떤 부위를 측정하는지 선택하는 물리적 버튼이나 시퀀스가 있다면, 전송 시 아래 ID를 사용해야 웹(ReportView)의 Heatmap UI와 정확히 1:1 매핑됩니다.

* `0x00` (0) : 이마 (Forehead)
* `0x01` (1) : 왼쪽 볼 (Left Cheek)
* `0x02` (2) : 오른쪽 볼 (Right Cheek)
* `0x03` (3) : 턱 (Chin)
* `0x04` (4) : 코 (Nose)
* `0x05` (5) : T존 (T-Zone)

## 3. 동작 시나리오 (Sequence)

1. **[사용자]** 웹 브라우저 접속 후 `[장비 연결]` 버튼 클릭.
2. **[웹 앱]** Chrome 블루투스 디바이스 선택 창 팝업.
3. **[사용자]** 기기 전원을 켜고 페어링 모드 진입. 웹 UI에서 기기 선택 후 페어링.
4. **[웹 앱]** Service UUID와 Characteristic UUID를 찾아 `startNotifications()` 호출하여 데이터 구독 시작.
5. **[장비]** 피부에 장비 접촉 후 스캔. 측정 완료 시 피드백 발생.
6. **[장비]** 측정 결과 패킷 전송 (예: 수분 45%, 유분 80% = `[0xAA, 0x01, 0x2D, 0x50, 0x00, Checksum]`).
7. **[웹 앱]** 패킷 수신 즉시 UI에 수분(45%)과 유분(80%)를 동시 업데이트.
8. (필요 부위 반복 측정)
9. **[사용자]** 웹 화면 하단 `[측정 완료]` 버튼 선택. 
10. **[웹 앱]** 블루투스 연결을 정리하고 측정 결과 기반 분석 리포트 화면 노출.

---

## 4. 특이사항 및 논의 필요 항목

* **기기의 부위 식별 방식:** 장비 자체에 부위를 선택하는 버튼이 있는지, 아니면 단순히 측정 버튼 1개만 있고 웹 화면의 지시에 따라(예: "이마를 대주세요") 순서대로 찍는 방식(Sequence 방식)인지에 따라 페이로드의 `Region ID` 취급 방법이 달라질 수 있습니다.
  * *만약 순서대로 찍는 방식이라면, 기계는 Value만 보내고 웹 프론트엔드 상태 머신이 해당 순서에 맞춘 부위 ID를 동적으로 매핑해야 합니다.* 

이 문서(초안)를 하드웨어 파트와 공유하신 후, 장비 개발 환경에 맞춰 수정/확정 부탁드립니다.
